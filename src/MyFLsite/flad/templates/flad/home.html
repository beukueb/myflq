Mission statement

FLAD is a simple database for forensic alleles, that offers permanent identifiers to any allele sequence. If a sequence is not yet in the database, a simple registration is sufficient to start adding alleles to FLAD.
FLAD does not provide information about the loci a sequence belongs to, or any other type of annotation. FLAD aims to be a simple identifier service, similar to what doi does for publication identifiers, to provide forensic analysis software with permanent identifiers for sequences.

Database download

A weekly backup is made from the database that is available for download (FLADbackup.tar.gz,verification hash). The verification hash is encrypted with the forensic.ugent.be certificate (public key). The origin of FLADbackup can be determined for quality control with the following commandline:

openssl dgst -sha256 -verify forensic.ugent.be.pub -signature FLADbackup.sha256 FLADbackup.tar.gz {% if user.userprofile.fladRequest and not user.userprofile.fladPriviliged %}

Testing FLAD

If you requested FLAD and are waiting for authorization, you can start using FLAX for testing and development. FLAX works exactly as FLAD, except that identifiers are not guaranteed to be permanent. FLAX identifiers start with 'T' instead of 'F'.
