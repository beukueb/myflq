Programmatory access

Retrieve sequence with a FLADid

https://forensic.ugent.be/flad/getseq/xml/{FLADid}

Example: https://forensic.ugent.be/flad/getseq/xml/FA001

Retrieve FLADid with a sequence

https://forensic.ugent.be/flad/getid/xml/{sequence}

Example: https://forensic.ugent.be/flad/getid/xml/ACTGACTAGACATGCGAC

Validate sequence and retrieve its FLADid

https://forensic.ugent.be/flad/validate/xml/{sequence}?user={username}&password={password}

Example: https://forensic.ugent.be/flad/validate/xml/ACTGACTAGACATGCGAC?user=jeffreys&password=1984

Unvalidate sequence or FLADid

If there are no other users validating this sequence it will be removed, otherwise it only removes the user's validation.

https://forensic.ugent.be/flad/unvalidate/{sequence}?user={username}&password={password}

Example: https://forensic.ugent.be/flad/unvalidate/ACTGACTAGACATGCGAC?user=jeffreys&password=1984

Plain text instead of xml

To get a simple byte string back consisting only of the requested information, use 'plain' instead of 'xml'. In this case, when providing a FLADid to retrieve a sequence, only the sequence present in the database will be send back and not its complementary strand.

Example: https://forensic.ugent.be/flad/getid/plain/ACTGACTAGACATGCGAC

Testing

For development and experimenting there is a parallel testing service FLAX. The API is identical, except that 'flad' needs to be replaced with 'flax' and that no authorization for validation is required. It is not possible to unvalidate alleles. Use only for testing.

Example: https://forensic.ugent.be/flax/validate/plain/ACTGACTAGACATGCGAC
